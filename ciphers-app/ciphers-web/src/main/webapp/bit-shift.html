About The Bit Shift Cipher

Bit Shift Ciphers, or in this case, Bit Rotation Ciphers, use the fact that characters are represented by their binary ASCII values in a computer. By manipulating those binary values by shifting them or rotating them by some number of bits the left or right, their value can be changed. Then, by shifting them the opposite direction (right or left respectively) by the same number of bits, the original value can be retained. The number of shifts is called the Shift Key.


Students of any Assembler course would be familiar with the concepts of bit shifting and bit rotations, and why it is generally better to use rotation shifting for this purpose.


In general, bit shifts will take the binary representation of a number, shift its bits in one direction and fill the empty spaces with zeros. Example: 1011 0110 shifted left one bit is 0110 1100. Notice the number of significant bits (1's) is NOT preserved: if a 1 is shifted, it gets removed from the binary value.


On the other hand, rotations will take the binary representation of a number and still shift the bits in one direction, but instead of filling the empty spaces with zeros, they are filled with the digit that was removed. Example: 0110 1100 rotated left one bit is 0110 1101. Notice the number of significant bits (1's) is retained here: this is why it's generally better to use rotations - the original value returns when the opposite operation is done on the new bit pattern.


Character sizes can be different depending on the system type (and sometimes the programming language). This algorithm assumes characters have a byte length (8 bits).


Since there are two rotation directions, this algorithm doesn't have specific enciphering and deciphering algorithms. Your recipient would need to know the direction of the enciphering shift (so they can reverse it for the deciphering shift), the shift key and the ciphertext to produce the correct plain text. The webpage shown after a message is enciphered will show you all of these.

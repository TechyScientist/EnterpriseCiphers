About the Shuffle Transposition Cipher

This cipher is based on Kattis problem Text Encryption (Kattis ID: textencryption), Summarized below:


  • The Kattis problem describes the deciphering algorithm as requiring the cipher text and a key value (n). The decipher algorithm starts at the firsr character of the cipher text, adds it to the plain text, skips (n - 1) characters, and adds it to the plaintext. This repeats until the end of the cipher string is reached, then is repeated starting with the second character in the cipher string (and so on) until all letters are used (i.e., the length of the plain text string is the same as the length of the cipher text string).
  • To encipher a string, a similar process is used. Starting from the first character, it is added to the cipher text, then (n - 1) characters are skipped, until the end of the string, repeating until all characters are used (i.e., the length of the cipher text matches the length of the message).
  • Without the shift key (n), the string is unreadable.
  • Due to slight differences in how the two algorithms need to work, they are not reversible! There is a separate enciphering and deciphering algorithm.
  • Note: This algorithm does not support using spaces. Spaces in the input string will automatically be removed, and the input string will automatically be converted to uppercase.
